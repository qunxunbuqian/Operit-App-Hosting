  ❓ 什么是「绑定」？

  app_hosting_v2.js（Operit 端）本身只是一个"遥控器"，
  它通过读写 config.json 来控制锁屏行为，但真正在屏幕上
  显示倒计时浮窗、锁屏密码键盘的，是运行在 AutoJS6 里的
  守护脚本 app_lock_v11.js。

  遥控器 → 改配置 → 发指令 → AutoJS6 执行守护脚本 → 锁屏
  ↑                    ↑                      ↑
  Operit 沙盒包        am start 命令          app_lock_v11.js

  所以「绑定」就是把守护脚本部署到手机上的这一步：

  ┌─────────────────────────────────────────┐
  │                                         │
  │  ① 在手机上创建文件夹：                   │
  │     /sdcard/脚本/                        │
  │                                         │
  │  ② 把 daemon/app_lock_v11.js 复制进去：   │
  │     /sdcard/脚本/app_lock_v11.js          │
  │                                         │
  │  ③ 在 AutoJS6 中运行一次该脚本             │
  │     （建议开启 AutoJS6「开机自启」）         │
  │                                         │
  └─────────────────────────────────────────┘

  ⚠️ 如果没有做这一步，app_lock_restart 会报错！

🔐 AI 修改密码安全指南
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ⛔ 永远不要直接传 defaultPassword 参数！
     调用参数会在 Operit UI 暴露明文。

  ✅ 正确流程（读后即焚）：
     1. AI 在脑中把密码做 base64 编码（如 1784 → MTc4NA==）
     2. 写入临时文件：
        echo "MTc4NA==" | base64 -d > /sdcard/Download/.app_lock_pw
     3. 调用 app_lock_edit_config 传 loadPasswordFromFile: "true"
     4. 工具读取密码后立即删除文件，无人可见
