     AI 修改密码时不再暴露明文——密码写入临时文件

     详见下方「AI 修改密码安全指南」。

  ✨ maskConfig 自动遮蔽
     所有配置输出中，密码字段自动显示为 "****"，
     防止敏感信息泄露。

  ✨ 兼容性保留
     与旧版 v1.2 完全兼容，config.json 格式不变。

🔧 前置依赖（缺一不可！）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ✅ AutoJS6 —— 必须安装
      下载地址：https://github.com/SuperMonster003/AutoJS6/releases

  ✅ 悬浮窗权限 —— 必须授予 AutoJS6
      设置 → 应用管理 → AutoJS6 → 显示悬浮窗/在其他应用上层显示

  ✅ 无障碍服务 —— 必须开启
      设置 → 辅助功能 → AutoJS6 → 开启无障碍服务

  ✅ Shizuku 或 Root 权限 —— 必须授予 Operit
      用于执行 am 命令启停守护脚本

📥 安装步骤
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  第 1 步：确认前置依赖全部就绪
  第 2 步：把「daemon/app_lock_v11.js」复制到手机
           → /sdcard/脚本/app_lock_v11.js
           （如果没有「脚本」文件夹，自己新建一个）
  第 3 步：把「config.json」复制到手机
           → /sdcard/Download/app_lock/config.json
           （如果没有 app_lock 文件夹，自己新建一个）
  第 4 步：打开 Operit，安装插件
           → 工具箱 → 包管理 → 右上角"手动安装"
           → 选择 app_hosting_v2.js
  第 5 步：启用「应用托管🔒 v2」插件
  第 6 步：通过插件工具添加要锁定的应用
           或直接编辑 config.json
  第 7 步：打开 AutoJS6，运行守护脚本
           → 打开 AutoJS6 → 点击 app_lock_v11.js → 运行
           （建议开启 AutoJS6 的"开机自启"功能）

🎮 使用方法
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  🔍 app_lock_get_config → 查看当前所有被锁应用及配置
  ✏️ app_lock_edit_config → 添加/修改/删除被锁应用
  🔄 app_lock_restart → 手动重启守护脚本
  🔄 app_lock_reset_config → 重置所有配置为默认值

  🔐 AI 修改密码安全指南（重要！）
     → 永远不要直接传 defaultPassword 参数——会在 Operit UI 暴露明文
     → 正确流程：
       1. 用 base64 编码密码（如 1784 → MTc4NA==）
       2. 写入临时文件：echo "MTc4NA==" | base64 -d > /sdcard/Download/.app_lock_pw
       3. 调用 app_lock_edit_config 传 loadPasswordFromFile: "true"
       4. 工具从文件读取密码后立即删除（读后即焚）

💡 小提示
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  • AutoJS6 建议开启"后台弹出界面"和"忽略电池优化"
  • 守护脚本默认路径为 /sdcard/脚本/app_lock_v11.js
  • 默认解锁密码为 1784（四位数字）
  • 所有应用可分别设置倒计时长和自动解锁延时
  • AI 修改密码请使用 loadPasswordFromFile，不要直接传参
